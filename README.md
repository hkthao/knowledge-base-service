# Knowledge Base Service

Index codebase + docs + issues into Neo4j (graph) and Qdrant (hybrid dense+BM25), expose `/search`
to n8n agent flows. See [`kb-full-plan-v4.md`](../mvp/kb-full-plan-v4.md) for the full design.

**Status:** weeks 1–4 of the 11-week plan are implemented:

- ✅ Tuần 1–2 — Foundation TS/JS + dual stores (Neo4j + Qdrant hybrid)
- ✅ Tuần 3 — Roslyn .NET service + C# semantic analysis
- ✅ Tuần 4 — Vietnamese descriptions (async queue) + Markdown doc indexing
- ⏳ Tuần 5 — Change management (git diff, ripgrep relink, repair job)
- ⏳ Tuần 6 — Search & graph expansion endpoints
- ⏳ Tuần 7–8 — Eval (golden set + iterate)
- ⏳ Tuần 9 — Hardening
- ⏳ Tuần 10–11 — n8n integration polish

## Quick start

```bash
cp .env.example .env

# Bring up infra (Neo4j + Qdrant + Ollama + Roslyn .NET service + Langfuse)
docker compose up -d neo4j qdrant ollama roslyn-service

# Pull embedding models (first run only)
docker compose exec ollama ollama pull nomic-embed-text
docker compose exec ollama ollama pull nomic-embed-code   # ~2GB

# Bootstrap stores: Neo4j constraints + Qdrant collections + SQLite state
python -m scripts.setup_neo4j
python -m scripts.setup_collections

# Index a repo (auto-detects .ts/.tsx/.js/.jsx and .cs files)
python -m scripts.initial_index --repo my-project --path /path/to/repo
```

For C# repos, the indexer auto-discovers each `.cs` file's owning `.csproj` —
no extra config required. To keep CALLS edges into internal NuGet packages
(symbols whose source lives in metadata), set:

```bash
INTERNAL_NS_PREFIXES=MyOrg.Internal,MyOrg.Shared
```

## Endpoints

```http
GET  /health                  -- Neo4j + Qdrant + Roslyn liveness
POST /index/file              -- idempotent re-index of one file
POST /index/repo              -- background full repo index
POST /index/docs              -- background full docs-dir index
POST /index/doc/file          -- index one Markdown / PDF / DOCX file
POST /index/delete            -- drop file from all collections
```

Vietnamese descriptions are generated by a separate worker so `/index/file`
never blocks on the LLM:

```bash
python -m scripts.desc_worker          # long-running drainer
python -m scripts.desc_worker --once   # process one batch, exit
```

`/search` and `/search/{callers,callees,co_changed}` land in week 6.

## Qualified names

Symbols are uniquely identified by `qualified_name` across the codebase:

```text
TypeScript:
  Function:   src/auth/validator.ts::validateUser
  Method:     src/auth/AuthService.ts::AuthService.login
  Class:      src/auth/AuthService.ts::AuthService

C# (uses namespace, not file_path — partial classes split across files
 still resolve to a single qualified_name):
  Method:     MyProject.Auth::AuthService.Login
  Class:      MyProject.Auth::AuthService
  Interface:  MyProject.Auth::IAuthService
```

CALLS edges carry a `confidence` score and `resolution_type`:

| Resolution                              | confidence |
|-----------------------------------------|------------|
| C# semantic (Roslyn)                    | 1.0        |
| TS same-file, name resolved             | 0.9        |
| TS cross-file, import resolved          | 0.7        |
| TS cross-file, name match heuristic     | 0.5        |
| Dynamic dispatch (`this[method]()`)     | (no edge)  |
